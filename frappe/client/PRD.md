# Frappe Client SDK

A type-safe, declarative API for defining and interacting with Frappe DocTypes
in Vue applications.

- **Type Safety**: Full TypeScript support with automatic inference.
- **Declarative**: Methods and controllers defined with simple metadata.
- **Reactive**: Automatic state management across the entire app.
- **Real-time**: Built-in WebSocket synchronization.

---

## 1. Initialization (Factory Pattern)

V3 moves away from global state. You create a project-specific data layer once
and export its utilities.

```ts
// src/client.ts
import { createClient } from 'frappe-ui/frappe'

export const { defineDoctype, client } = createClient({
  baseUrl: 'https://mysite.com',
  realtime: true, // Enable real-time sync globally
})
```

## 2. DocType Definition (Generated)

The blueprint defines the schema and methods. This is auto-generated by a
script.

```ts
// src/doctypes.ts (Generated)
import { defineDoctype } from '~client' // ~client is aliased to src/client.ts via vite.config

export interface ToDoInterface {
  name: string
  description: string
  status: 'Open' | 'Closed'
  priority: number
  assigned_to?: string
  owner: string
  modified: string
  creation: string
}

export const ToDo = defineDoctype<ToDoInterface>()('ToDo', {
  // Document-level methods (operate on a specific instance)
  docMethods: {
    setStatus: {
      method: 'set_status',
      httpMethod: 'POST',
      args: (params: { status: 'Open' | 'Closed' }) => params,
    },
    assignTo: {
      method: 'assign_to',
      httpMethod: 'POST',
      args: (params: { user: string; description?: string }) => params,
    },
  },
  // Controller-level methods (operate at DocType level)
  controllerMethods: {
    sendEmail: {
      method: 'send_email',
      httpMethod: 'POST',
      args: (params: { names: string[] }) => params,
    },
  },
})
```

## 3. Usage Examples

### getDoc

```ts
import { ToDo } from '@/doctypes'

let props = defineProps<{
  todoId: string
}>()

let todo = ToDo.getDoc(() => props.todoId) // takes MaybeRefOrGetter<string>, refetches on change
todo.doc // reactive doc object
todo.loading // boolean, true when request is in-flight
todo.error // if fetch fails, todo.error contains error message sent by server
todo.reload() // refetch the doc
```

#### Standard document methods

setValue method is available on `todo` for updating single/multiple fields

```ts
// field updates
await todo.setValue.submit({ description: 'New description' })
todo.setValue.loading // true when request is in flight
todo.setValue.error // error when request fails

// optimistically update UI
// will set todo.doc.status = 'Closed' immediately, make the fetch call,
// update the todo.doc based on success response, reverts todo.doc to original on error
todo.setValue.submit({ status: 'Closed' }).optimistic()
```

#### Custom Document Methods

`docMethods` are also available on `todo` defined by defineDoctype in
doctypes.ts

```ts
// Type-safe method calls
await todo.setStatus.submit({ status: 'Closed' })
todo.setStatus.loading
todo.setStatus.error

// Method with mixed parameters
await todo.assignTo.submit({ user: 'faris@frappe.io', description: 'Priority' })
todo.assignTo.loading
todo.assignTo.error
```

### getList

```ts
import { ToDo } from '@/doctypes'

let statusFilter = ref('Open')
let orderBy = ref('creation desc')

let todos = ToDo.getList({
  fields: ['name', 'description', 'status', 'priority'],
  filters: {
    // filter values can take MaybeRefOrGetter so that when the value changes, list gets refetched automatically
    status: statusFilter,
    assigned_to: 'user@example.com', // can take static values as well
  },
  orderBy: orderBy, // similar to filter keys, can be MaybeRefOrGetter<string>
  start: 0, // MaybeRefOrGetter<number>
  limit: 20, // MaybeRefOrGetter<number>
  pageMode: 'append', // append | replace, used for pagination
})

todos.data // reactive array of todo objects
todos.loading // boolean, true when request is in-flight
todos.hasNextPage // boolean, true when next page is available (information sent from server)
todos.hasPreviousPage // boolean
todos.next() // next page (increment the start param by "limit" and refetch), mutate todos.data based on pageMode
todos.previous() // previous page
todos.reload() // re fetch
todos.error // if fetch fails, todos.error contains the error message sent by server
```

### New Document (Draft Mode)

Creates a local-only reactive document for "Create" forms.

```ts
const todo = ToDo.newDoc({
  status: 'Open', // initialize with defaults
})
todo.doc.description = 'Draft task'

await todo.insert.submit()
// → POST /api/v2/document/ToDo { ... }
```

### Custom Controller Methods

Custom controller methods are available on the ToDo object defined by
`controllerMethods` key in defineDoctype

```ts
// typesafe controller method call
const sendEmail = ToDo.sendEmail.create()
await sendEmail.submit(['todo-1', 'todo-2'])
```

### Bulk Operations

```ts
// Deletion
await ToDo.bulkDelete.submit(['todo-1', 'todo-2'])

// Field Update (Per-item changes)
await ToDo.bulkUpdate.submit([
  { name: 'todo-1', status: 'Closed' },
  { name: 'todo-2', status: 'Closed' },
  { name: 'todo-3', description: 'Updated description' },
])
// Should automatically update the todo cache so that any "lists" or "documents" that refer to any of these todos get updated automatically
```

### File Uploads

Specialized API for handling file attachments with progress tracking.

```ts
import { File } from 'frappe-ui/frappe'

let file = File.new({
  file: myFileBlob, // Blob
  doctype: 'ToDo',
  name: 'id',
  fieldname: 'attachments', // Optional: attach to specific field
  isPrivate: true, // Optional: defaults to true
  folder: 'Home/Attachments', // Optional: destination folder
})

file.upload() // initiate upload
file.progress // Reactive upload progress (0-100)
file.doc // Server success response (the File document)
file.error // Server error response
```

#### DocInfo & Permissions

Metadata like permissions, assignments, and attachments are separated into an
on-demand fetch to keep `getDoc` light.

```ts
const info = todo.getDocInfo() // Returns a reactive operation object

// Fetch on-demand
await info.submit()

info.data.permissions // { read: 1, write: 1, ... }
info.data.attachments // array of file objects
info.data.assignments // list of users assigned
info.data.comments // list of comments/communications
info.loading // boolean
```

## 5. Real-time Synchronization (Socketio)

When `realtime: true` is set during initialization, the data layer automatically
reacts to realtime events.

1. Listen for `doc_update` and `doc_rename`.
2. Updates result in an automatic call to `syncFromDocs(payload.docs)`.
3. UI components (Doc/List) watching these docs update **instantly**.

### Realtime API

```ts
// Listen for specific changes
ToDo.onUpdate((name) => {})

ToDo.onRename((name, oldName) => {})
```

## Backend Requirements

```python
class ToDo(Document):
    @frappe.whitelist()
    def set_status(self, status):
        self.status = status
        self.save()
        return self

@frappe.whitelist()
def send_email(names):
    return {"sent": len(names)}
```
